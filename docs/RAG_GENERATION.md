# Phase 6 — Grounded RAG Generation

Phase 6 converts the Phase 5 retrieval results into a safe, cited answer using
`gemini-3.1-flash-lite`.

## Pipeline

```text
question
  -> input length validation
  -> prompt-injection guard
  -> UUD 1945 domain guard
  -> Gemini query embedding
  -> Neon hybrid retrieval
  -> bounded context builder
  -> Gemini grounded answer
  -> deterministic sources from database chunks
```

Out-of-domain and prompt-injection requests stop before query embedding,
retrieval, and answer generation. A valid query with no retrieved context stops
before answer generation and returns `no_context`.

## Statuses

- `answered`: Gemini answered from retrieved UUD context.
- `rejected`: the question is outside UUD 1945 or contains prompt injection.
- `no_context`: no chunk passed retrieval.
- `error`: reserved for the API layer when a provider or internal operation
  fails.

## Grounding controls

- The context contains at most 5 chunks and 6,000 characters.
- Temperature is `0.1`.
- Output is capped at 500 tokens.
- The system instruction forbids invented facts, quotations, articles, or
  sources.
- Source citations are built from Neon metadata, not generated by the model.
- API keys, full vectors, and full documents are never logged.

## Test

```powershell
cd frontend
npm run test:rag
npm run evaluate:rag
```

The live evaluation covers:

- an exact Pasal/ayat question;
- a conceptual UUD question;
- an out-of-domain question;
- a prompt-injection attempt.

The first two invoke Gemini. The latter two must return `rejected` without
calling Gemini.
